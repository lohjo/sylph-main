# Design Context

## Users

Mixed audience: hackathon judges, university admissions officers, technical recruiters, research mentors, fellow student builders, policy/civic-tech folk who saw John Ray present. Browsing context: daytime, desktop or phone, often skimming for credibility signals before deciding to reach out. They want to know in 30 seconds: _is this person serious, what have they shipped, can I trust the technical depth?_

Job to be done: confirm John Ray is a credible builder + researcher worth engaging with, then go deep on whichever artifact (project, paper, essay) drew them in.

## Brand Personality

Three words: **rigorous, warm, multilingual**.

Voice: a researcher who ships. Confident without strutting. Cites primary sources and tags equations correctly. Switches register from breakthrough-curve modelling to Vietnamese school-renovation reflection without sounding like two different sites.

Emotional goals: trust → curiosity → "I want to talk to this person." Avoid: showing off, hustler-energy, generic-developer-portfolio neutrality.

## Aesthetic Direction

**Reference:** raphaelsalaja.com — quiet, editorial, single-column, almost manuscript-like. Sylph template (which this project forks) inherits that.

**Anti-references:**

- Generic Vercel-template developer portfolios (gradient hero, rotating tech-stack icons, "I love coffee and clean code" copy).
- Over-designed agency portfolios (parallax hero, 3D blobs, scroll-jacking).
- Notion-as-website / Linktree-as-website.

**Theme:** light by default (daytime professional context), with high-quality dark mode for evening / accessibility — both must be intentionally tuned, not auto-inverted.

**Direction:** _Editorial scientific_. Manuscript-grade reading experience. Single column dominated by content. Subtle deep-teal accent (#0f4c5c-ish, expressed as OKLCH) referencing both the polymer/lab work and Singapore's quiet-civic-design tradition. Neutrals tinted toward the teal hue at chroma ~0.005. Type pairs a warm scientific serif (display) with a clean modern grotesque (body). Generous, varied vertical rhythm. The signature memorable element is the inline `<BreakthroughCurve />` component — a real, annotated lab chart embedded in a research post. That moment is what someone remembers.

**Typography (locked, off the impeccable banned list):**

- Display / headings: **Source Serif 4** (variable, OpenType ligatures + small caps) — academic warmth without the Fraunces/Playfair/Instrument-Serif cliché.
- Body: **Geist** — restrained modern grotesque, distinct from the Inter/DM Sans monoculture.
- Mono / inline code: **JetBrains Mono** — keeps code blocks readable and is more distinctive than Space Mono / Plex Mono.

Type scale: modular ratio 1.25, fluid via `clamp()` on display sizes, fixed `rem` on body. Body line length capped at 68ch. Headings get extra space _above_ (varied rhythm), not equal margins.

**Color (OKLCH):**

- Surface base (light): `oklch(0.98 0.005 200)`
- Ink (light): `oklch(0.18 0.01 220)`
- Surface base (dark): `oklch(0.16 0.01 220)`
- Ink (dark): `oklch(0.92 0.005 200)`
- Accent (both): `oklch(0.50 0.07 195)` — deep muted teal, used sparingly per 60-30-10 rule (links, in-text emphasis, chart fit-line).

## Design Principles

1. **Content is the chrome.** No decorative borders, no glass cards, no gradient text. Every visual element justifies itself by carrying meaning.
2. **One memorable moment per post.** Project pages have a hero artifact (CAD render, architecture diagram); research has the live breakthrough chart; writing has a single pulled quotation. Don't dilute.
3. **Vary the rhythm.** Tight stacks for metadata, generous space around section transitions. Resist applying the same padding everywhere.
4. **Code blocks are first-class.** Syntax-highlighted, copy-able, language-labeled. Code is part of the writing, not an aside.
5. **Bilingual-ready surfaces.** Spacing and type sizes hold up if a phrase is rendered in Mandarin or Vietnamese. Don't pixel-tune for English-only.
